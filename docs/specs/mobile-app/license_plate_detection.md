# License Plate Detection Model — Phase 1

## Purpose

Define the training data sources, training process, and export pipeline for the YOLOv8-nano license plate detection model used in the mobile app.

## Model Overview

| Property | Value |
|---|---|
| Architecture | YOLOv8-nano (3.2M parameters) |
| Base weights | COCO-pretrained (`yolov8n.pt`) |
| Training approach | Fine-tune from COCO (not from scratch) |
| Output class | Single class: `license-plate` |
| Export targets | Core ML (`.mlmodel`) for iOS, TFLite (`.tflite`) for Android |

## Training Data

### Primary Source: HuggingFace License Plate Detection

- **URL**: https://huggingface.co/datasets/keremberke/license-plate-object-detection
- **Size**: 8,823 images (6,176 train / 1,765 val / 882 test)
- **Classes**: `license-plate` (single class)
- **Format**: COCO (converted to YOLO format by `train.py`)
- **Origin**: Large-scale license plate detection dataset

This dataset provides substantially more training data than earlier alternatives, leading to better detection accuracy.

### Data Augmentation

YOLOv8 applies its built-in augmentation pipeline during training (mosaic, HSV shift, flip, scale, etc.).

## Training Pipeline

### Prerequisites

```
pip install ultralytics datasets
```

### Steps

1. **Download dataset** from HuggingFace, convert from COCO to YOLO format (automated by `train.py`)
2. **Train** YOLOv8-nano with fine-tuning:
   ```
   yolo detect train \
     model=yolov8n.pt \
     data=dataset.yaml \
     epochs=100 \
     imgsz=640 \
     batch=16 \
     name=plate-detector-v1
   ```
3. **Validate** on held-out test set — target metrics:
   - mAP@0.5 ≥ 0.85
   - mAP@0.5:0.95 ≥ 0.60
   - Inference time < 30ms on desktop GPU (proxy for mobile performance)
4. **Export** to mobile formats:
   ```
   yolo export model=best.pt format=coreml    # iOS
   yolo export model=best.pt format=tflite    # Android
   ```

### Validation Criteria

The model MUST meet these thresholds before being bundled into the app:

| Metric | Minimum | Target |
|---|---|---|
| mAP@0.5 | 0.80 | 0.90 |
| Precision | 0.80 | 0.90 |
| Recall | 0.75 | 0.85 |
| False positive rate | < 10% | < 5% |
| Inference (iPhone 12) | < 50ms | < 30ms |
| Inference (Pixel 6) | < 50ms | < 30ms |

If Phase 1 metrics fall short, proceed to Phase 2 (additional data sources) before shipping.

## Export Details

### iOS (Core ML)

```python
from ultralytics import YOLO
model = YOLO("best.pt")
model.export(format="coreml", nms=True, imgsz=640)
```

Output: `best.mlpackage` — bundle in Xcode project under `Models/`.

### Android (TFLite)

```python
from ultralytics import YOLO
model = YOLO("best.pt")
model.export(format="tflite", imgsz=640)
```

Output: `best_float32.tflite` — bundle in Android project under `assets/`.

## Model Versioning

- Models are stored in the repo under `models/` (gitignored if >100MB, use Git LFS otherwise)
- Each model release is tagged: `plate-detector-v1.0`, `plate-detector-v1.1`, etc.
- The `models/CHANGELOG.md` file tracks: training data used, augmentation settings, validation metrics, export formats

## Constraints

- C-1: Model must be < 10 MB to keep app bundle size reasonable (YOLOv8-nano is ~6.2 MB)
- C-2: Single class detection only (`license-plate`) — vehicle detection is not needed
- C-3: Input resolution fixed at 640x640 (standard YOLO input, frames are resized before inference)

## Implementation Plan

### Project Structure

```
models/
├── Makefile                      # Build/export/deploy targets
├── training/
│   ├── train.py                  # Training script (download data, train, validate, export)
│   ├── detect.py                 # Run detection on images/video for testing
│   ├── evaluate.py               # Evaluate model against validation criteria
│   ├── requirements.txt          # ultralytics, datasets
│   └── dataset.yaml              # YOLO dataset config (generated by train.py)
├── exports/
│   ├── plate_detector.mlmodel    # Core ML export (copied to ios/CamerasApp/Models/)
│   └── plate_detector.tflite     # TFLite export (copied to android/app/src/main/assets/)
└── CHANGELOG.md                  # Model version history with metrics
```

### Implementation Order

| Step | Description |
|---|---|
| 1 | Create `models/training/train.py` — downloads HuggingFace dataset, trains YOLOv8-nano, validates |
| 2 | Run training, evaluate against validation criteria |
| 3 | If metrics pass → export to Core ML and TFLite |
| 4 | Copy exports to iOS and Android project asset directories |
| 5 | Record metrics and config in `models/CHANGELOG.md` |
| 6 | Test inference on-device (both platforms) to validate latency and accuracy |
